# 测试问题

- 1. 问题: 在画板中选中按钮组件, 并切换到属性面板到EVENTS后, 切换选中组件选择文本组件, 但是此时属性面板还是显示EVENTS,问题是文本的属性面板没有EVENTS.
     期望: 切换选中组件选择文本组件, 属性面板显示文本的属性配置,所有组件切换选中时, 右侧属性面板都展示PROPS

2. 问题: 比如选中 MODAL组件时, 属性面板中没有展示完成的组件id
   期望: 属性面板展示完整的组件id,此外优化组件id的命名, 采用规则: 组件id = 组件名 + '\_' + 在当前编辑页面中出现的排序
3. 问题: 编辑页点击 TREE 无效果
   期望: 编辑页点击 TREE 展示组件树
4. 问题: 组件树现在交互只支持点击,需要扩展交互
   期望: 1.支持在组件树上进行拖拽排序, 拖拽效果同步在画板中更新; 2.组件树上支持删除组件
5. 问题: 左侧组件可以拖到到画板中的container容器中, 但是画板中的组件无法拖动到container容器中
   期望: 画板中的组件可以拖动到container容器中
6. 问题：按钮在属性面板中配置了events的action,在画板中点击按钮会触发配置的action。
   期望: 画板中的按钮点击是无效果的，只有预览时才有效果。
7. 问题：编辑页进行修改操作后离开编辑页面或者刷新页面时，无任何保存提示
   期望：编辑页进行修改操作后离开编辑页面或者刷新页面时，需要弹框提示
